# Factory Method Pattern - Java Bilingual Tutorial

A short software-design tutorial that introduces the Factory Method pattern in
Java using a very small geometry example. The only products are `Circle` and
`Triangle`. The main point is simple: the client asks a `Factory` to
`create()` a `Shape`, and each concrete factory knows how to build one kind of
shape.

The same scene sources render to both landscape and portrait, so the video works
for short videos and presentation slides. Narration text and Manim scene sources
are generated by the configured `ai_cli` from the descriptions below. Keep every
scene clean, visual, and minimal. Use semantic colors: DANGER for hardcoded
creation, OK for the Factory Method design, HIGHLIGHT for the current line or
object, PRIMARY for titles, and ACCENT for labels. All code examples should be
written in Java. Use the role name `Factory`, not `Creator`.

## Scene: 01_pengantar / Pengantar

**file:** scene_01_pengantar.py
**fallback_duration:** 10

### description
Title card. The first scene must display the main title exactly as
`Factory Method pattern`. Introduce it as a creational design pattern whose
intent is to let subclasses decide which object to create. Keep the screen very
simple: one large title, one short subtitle, and two shape icons: Circle and
Triangle.

## Scene: 02_masalah / Masalah

**file:** scene_02_masalah.py
**fallback_duration:** 16

### description
Problem statement built around a Java geometry app. A client class needs shapes,
but it directly creates concrete classes:
`new Circle(center, radius)` and `new Triangle(a, b, c)`. Show the client with
two DANGER arrows pointing to the concrete constructors. Keep the message short:
the client is now responsible for knowing how each shape is created.

## Scene: 03_pendekatan_naif / PendekatanNaif

**file:** scene_03_pendekatan_naif.py
**fallback_duration:** 18

### description
Show a naive central factory with one method:
`create(String type)`. The code uses `if (type.equals("circle")) return new
Circle(...); else return new Triangle(...);`. Use DANGER color on the `if/else`.
Call out only one issue: every new shape means editing this same method.

## Scene: 04_refactor_factory_method / RefactorFactoryMethod

**file:** scene_04_refactor_factory_method.py
**fallback_duration:** 22

### description
Introduce Factory Method as the solution. Build the design step by step in OK
color: `Shape`, `Circle`, `Triangle`, and an abstract `Factory` with one
method: `create(): Shape`. Show `CircleFactory` returning `new Circle(...)` and
`TriangleFactory` returning `new Triangle(...)`. Emphasize the key line:
`Shape shape = factory.create();`. Also make clear that it is not
`shape.create()`.

## Scene: 05_diagram / Diagram

**file:** scene_05_diagram.py
**fallback_duration:** 18

### description
Display the UML class diagram of the Factory Method pattern using the
uml_class_box helper. Include a `Shape` interface with `+ draw(): void`,

`CircleFactory` and `TriangleFactory`. Keep the diagram small enough to fit on
one slide. Annotate the key idea: factories create shapes.

## Scene: 06_perbandingan_kode / PerbandinganKode

**file:** scene_06_perbandingan_kode.py
**fallback_duration:** 22

### description
Side-by-side comparison of Java code before and after applying Factory Method,
using code_card panels. On the left side in DANGER, show:
`if (type.equals("circle")) return new Circle(...);`. On the right side in OK,
show:
`class CircleFactory extends Factory { Shape create() { return new
Circle(...); } }`. Highlight the simple difference: before, one method chooses
everything; after, each factory creates one shape.

## Scene: 07_kesimpulan / Kesimpulan

**file:** scene_07_kesimpulan.py
**fallback_duration:** 14

### description
Conclusion. Summarize the Factory Method intent: let factory subclasses decide
which object to create. Recap the benefits in short phrases: no messy `if/else`,
client depends on `Factory`, and each factory creates one product. End with
